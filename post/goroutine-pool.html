GoRoutinePool

一个基于 Go 的轻量级协程池实现，通过预创建固定数量的 Worker 协程并复用，避免频繁创建/销毁 goroutine 带来的调度开销，同时有效控制并发数量。

特性

  • 固定 Worker 数量 — 预创建指定数量的工作协程，任务在 Worker 间轮转复用
  • 任务队列缓冲 — 支持带缓冲的任务队列，队列满时 Submit 超时返回错误
  • 结果通道回调 — 每个 Task 携带独立的 Result channel，支持异步获取执行结果
  • Panic 恢复 — Worker 内置 recover，单个任务 panic 不会拖垮整个池
  • 优雅关闭 — Stop 依次关闭任务提交、分发通道、Worker 协程，确保已提交任务执行完毕
  • 并发安全 — 使用 sync.Mutex、sync.Once、atomic 保证状态操作的线程安全

快速开始

安装

go get github.com/yourname/GoRoutinePoolbash

基本用法

package main

import (
    "fmt"
    "strconv"
    goroutine_pool "github.com/yourname/GoRoutinePool"
)

func main() {
    // 创建协程池：4 个 Worker，任务队列缓冲 10
    pool := goroutine_pool.NewPool(4, 10)
    pool.Start()

    // 定义任务函数
    greet := func(args ...interface{}) interface{} {
        name := args[0].(string)
        return "Hello, " + name + "!"
    }

    // 构造并提交任务
    task := goroutine_pool.Task{
        Execute: greet,
        Args:    []interface{}{"chanchan"},
        Result:  make(chan interface{}, 1),
    }

    if err := pool.Submit(task); err != nil {
        fmt.Println("submit failed:", err)
        return
    }

    // 从 Result channel 获取结果
    result := <-task.Result
    fmt.Println(result) // Hello, chanchan!

    // 关闭协程池
    pool.Stop()
}go

API

NewPool(maxWorkerNum, maxQueueSize uint32) *Pool

创建协程池。maxWorkerNum 为 Worker 数量，maxQueueSize 为任务队列缓冲大小。

pool.Start()

启动协程池，开始从任务队列中取出任务分派给空闲 Worker。

pool.Submit(task Task) error

提交任务。若队列满，最多阻塞 1 秒后返回 ErrQueueFull；池已关闭则返回 ErrPoolClosed。

pool.Stop()

优雅关闭。先拒绝新任务提交，再等待已提交任务执行完毕，最后回收所有 Worker。可安全多次调用（内部 sync.Once 保证只执行一次）。

Task 结构体

字段 类型 说明
Execute func(args ...interface{}) interface{} 任务执行函数
Args []interface{} 传递给 Execute 的参数
Result chan interface{} 结果通道，任务完成后写入并关闭
Completed bool 任务是否执行完成

错误

错误 说明
ErrQueueFull 任务队列已满，Submit 超时
ErrPoolClosed 协程池已关闭，拒绝提交

架构

            Submit(task)
                │
                ▼
          ┌───────────┐
          │ taskChan   │  ◄── 带缓冲任务队列
          └─────┬─────┘
                │  Start() 分派
    ┌───────────┼───────────┐
    ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│ Worker1 │ │ Worker2 │ │ WorkerN  │   ◄── 固定数量
└─────────┘ └─────────┘ └─────────┘
    │           │           │
    ▼           ▼           ▼
task.Result  task.Result  task.Result   ◄── 调用方读取text

工作流程：

  1. NewPool 创建指定数量的 Worker，每个 Worker 启动一个 goroutine 等待任务
  2. 空闲 Worker 放入 idleWorkers channel
  3. Submit 将任务写入 taskChan
  4. Start 协程从 taskChan 取任务，从 idleWorkers 取空闲 Worker，分派任务
  5. Worker 执行完毕后将自身放回 idleWorkers，任务结果写入 Result channel
  6. Stop 关闭 taskChan → 关闭 stopChan 通知 Worker 退出 → wg.Wait() 等待全部完成

关闭顺序

Stop() 的关闭逻辑保证了已提交任务不会丢失：

  1. isStopped = true — 拒绝新的 Submit
  2. close(taskChan) — 不再接收新任务，Start 将队列中剩余任务分派完毕后退出
  3. close(stopChan) — 通知所有 Worker 停止等待
  4. wg.Wait() — 等待所有 Worker 执行完当前任务
  5. close(idleWorkers) — 清理空闲队列
